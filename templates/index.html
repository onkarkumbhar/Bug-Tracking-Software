Project Description

Software testers and Software devlopers needs centralized bug tracking and updating software.
where company can host their own server without risking data breach and can have internal structure.
Databases used for this project are based on minimal requirements.
This website contains:-

  • Employee login
  • Employee Register
  • Create Bug
  • Update Status of bug

Database Schema

            CREATE TABLE Company(
                company_id VARCHAR(10),
                company_name VARCHAR(40),
                company_email VARCHAR(30),
                employee_count int,
                primary key(company_id)
            );

            CREATE TABLE Models(
                model_id VARCHAR(10),
                model_name VARCHAR(40),
                company_id VARCHAR(10),
                bug_count int,
                primary key(model_id),
                foreign key (company_id) references Company(company_id)
            );

            CREATE TABLE Employee(
                employee_id VARCHAR(10),
                employee_name VARCHAR(40),
                employee_email VARCHAR(30),
                company_id VARCHAR(10),
                primary key(employee_id),
                foreign key(company_id) references Company(company_id)
            );

            CREATE TABLE Bug_Track(
                bug_id VARCHAR(10),
                bug_steps VARCHAR(300),
                bug_link VARCHAR(100),
                bug_status VARCHAR(20),
                company_id VARCHAR(10),
                employee_id VARCHAR(10),
                model_id VARCHAR(10),
                primary key(bug_id),
                foreign key(company_id) references Company(company_id),
                foreign key(employee_id) references Employee(employee_id),
                foreign key(model_id) references Models(model_id)
            );
